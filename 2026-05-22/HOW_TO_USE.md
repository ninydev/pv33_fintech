# Руководство по работе с dApp сетью (через cURL)

В этом руководстве показано, как взаимодействовать с вашей локальной P2P-сетью блокчейна, состоящей из 4-х нод (Sunny, Jonny, Gov, Bank), используя утилиту командной строки `curl`.

Предполагается, что вы уже запустили сеть с помощью скрипта `./start_nodes.sh`.

---

## 📋 Адреса нод в нашей сети

- **Sunny Node:** `http://localhost:3001`
- **Jonny Node:** `http://localhost:3002`
- **Gov Node:**   `http://localhost:3003`
- **Bank Node:**  `http://localhost:3004`

---

## 1. Чтение цепочки блоков (GET /blocks)

Чтобы посмотреть текущее состояние блокчейна на любой из нод, отправьте GET-запрос на эндпоинт `/blocks`.

**Пример: Смотрим цепочку на ноде Sunny (порт 3001)**
```bash
curl -X GET http://localhost:3001/blocks
```

*Сразу после запуска сети вы увидите массив из одного элемента — Genesis-блока.*

---

## 2. Добавление нового блока (POST /mineBlock)

Чтобы добавить новые данные в блокчейн, нужно отправить POST-запрос с JSON-телом на эндпоинт `/mineBlock`. Эта нода создаст блок и разошлет его всем остальным.

**Пример: Добавляем транзакцию через ноду Bank (порт 3004)**
```bash
curl -X POST http://localhost:3004/mineBlock \
     -H "Content-Type: application/json" \
     -d '{"data": "Перевод 100 монет от Alice к Bob"}'
```

*В ответ вы получите JSON с данными только что созданного блока.*

---

## 3. Проверка синхронизации сети (dApp в действии!)

Вся суть распределенного приложения (dApp) в том, что данные синхронизируются. Давайте это проверим.

1. **Добавьте блок через ноду Jonny (3002):**
   ```bash
   curl -X POST http://localhost:3002/mineBlock \
        -H "Content-Type: application/json" \
        -d '{"data": "Jonny выпустил новый смарт-контракт"}'
   ```

2. **Проверьте цепочку на совершенно другой ноде, например, Gov (3003):**
   ```bash
   curl -X GET http://localhost:3003/blocks
   ```
   *Вы увидите, что нода Gov имеет блокчейн, содержащий и Genesis-блок, и блок с транзакцией от Alice к Bob (если вы выполнили шаг 2), и новый блок от Jonny. Магия P2P сработала!*

---

## 4. Проверка подключенных пиров (GET /peers)

Вы можете посмотреть список ID веб-сокетов, подключенных к конкретной ноде.

**Пример: Смотрим соединения ноды Sunny (порт 3001)**
```bash
curl -X GET http://localhost:3001/peers
```
*Так как скрипт запуска связывает ноды последовательно (Jonny -> Sunny, Gov -> Sunny & Jonny, Bank -> All), вы увидите, что у Sunny несколько активных соединений.*
