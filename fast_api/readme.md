# ArtiGame Web3 Backend

Цей проект є backend-частиною на Python (FastAPI) для взаємодії зі смарт-контрактом `ArtiGame` у локальній мережі Ethereum (Hardhat).

## Важливе зауваження щодо безпеки ⚠️

У цьому проекті для цілей тестування та демонстрації **приватні ключі гравців передаються в тілі HTTP-запиту**. Це зроблено **виключно** для того, щоб показати, як бекенд може формувати та підписувати транзакції від імені різних користувачів у локальній тестовій мережі (Hardhat).

**У РЕАЛЬНОМУ ПРОЕКТІ (Production) РОБИТИ ТАК СУВОРО ЗАБОРОНЕНО!**
Приватні ключі ніколи не повинні залишати пристрій користувача. У реальних Web3-додатках (dApps) бекенд лише читає дані, а всі транзакції підписуються на стороні клієнта за допомогою криптогаманців (наприклад, MetaMask), і бекенд отримує лише хеш вже підписаної та відправленої транзакції або просто "слухає" події (events) контракту з блокчейну.

## Смарт-Контракт `ArtiGame`

Смарт-контракт написаний на Solidity і реалізує такі можливості:
- **Мінтер/Власник**: Користувач, який задеплоїв контракт, має права адміністратора.
- **Ковалі (Blacksmiths)**: Спеціальні ролі, яким адміністратор дозволив створювати (крафтити) предмети.
- **Предмети (Artifacts)**: Унікальні NFT-подібні предмети з характеристиками (атака, захист) та посиланням на зображення в IPFS.
- **Інвентар**: Кожен предмет прив'язаний до адреси конкретного гравця.
- **Торгівля**: Гравці можуть передавати свої предмети іншим, виставляти їх на продаж та купувати предмети інших гравців за криптовалюту (ETH/Wei).

## API Ендпоінти

Додаток реалізує REST API для взаємодії з контрактом. Нижче наведено опис кожного методу.

### Адміністративні методи (Викликаються системою)

Ці методи підписуються приватним ключем адміністратора, який "зашитий" у код (для локальної мережі).

*   `POST /admin/register-blacksmith`
    *   **Опис**: Реєструє вказану адресу як коваля, надаючи їй право створювати предмети.
    *   **Параметри**: `blacksmith_address` (адреса).
*   `POST /blacksmith/craft`
    *   **Опис**: Створює новий артефакт і одразу кладе його в інвентар вказаного гравця.
    *   **Параметри**: `name` (назва), `damage` (шкода), `defense` (захист), `ipfs_hash` (посилання на картинку), `player_address` (адреса гравця, якому видається предмет).

### Читання даних (Не потребує підпису транзакцій, безкоштовно)

*   `GET /artifacts/all`
    *   **Опис**: Повертає список всіх предметів, які будь-коли були створені в грі.
    *   **Поля у відповіді**: ID, назва, шкода, захист, IPFS хеш, хто створив (коваль), **поточний власник**, **ціна** (якщо предмет виставлено на продаж).
*   `GET /artifacts/player/{player_address}`
    *   **Опис**: Повертає список предметів, якими зараз володіє вказаний гравець.

### Дії гравців (Вимагають приватного ключа гравця)

Ці методи змінюють стан блокчейну і потребують оплати за газ (Gas).

*   `POST /player/transfer`
    *   **Опис**: Безпечна передача предмета іншому гравцеві (подарунок).
    *   **Параметри**: 
        *   `private_key`: Приватний ключ того, хто зараз володіє предметом (щоб підписати дозвіл).
        *   `artifact_id`: ID предмета.
        *   `to_address`: Адреса того, кому передається предмет.
    *   **Солідіті**: Викликає `transferArtifact(_to, _artifactId)`.
*   `POST /player/add-for-sale`
    *   **Опис**: Гравець виставляє свій предмет на продаж.
    *   **Параметри**:
        *   `private_key`: Приватний ключ власника.
        *   `artifact_id`: ID предмета.
        *   `price_wei`: Ціна продажу у wei (найдрібніша частка ефіру. 1 ETH = 10^18 wei).
    *   **Солідіті**: Викликає `addForSale(_artifactId, _price)`.
*   `POST /player/buy`
    *   **Опис**: Гравець купує предмет, який раніше був виставлений на продаж.
    *   **Параметри**:
        *   `private_key`: Приватний ключ покупця (з його балансу спишуться кошти).
        *   `artifact_id`: ID предмета, який він хоче купити.
    *   **Особливість**: Метод автоматично дізнається ціну предмета в смарт-контракті і відправляє точну суму `value_wei` у транзакції.
    *   **Солідіті**: Викликає `payable` функцію `buyArtifact(_artifactId)`.

## Запуск проекту

1.  Переконайтеся, що запущено локальну ноду Hardhat (`npx hardhat node`).
2.  Задеплойте контракт `ArtiGame` у мережу Hardhat і скопіюйте його адресу (Contract Address).
3.  Оновіть змінні `CONTRACT_ADDRESS` та `ADMIN_PRIVATE_KEY` у файлі `server.py` відповідно до результатів деплою.
4.  Запустіть сервер FastAPI (наприклад, за допомогою `uvicorn server:app --reload`).
5.  Відкрийте документацію Swagger UI у браузері (зазвичай `http://localhost:8000/docs`), де ви зможете зручно тестувати всі методи.
